哈希

哈希表

哈希（Hash） 是一种映射思想，规定存在一个唯一的哈希值与键值之前建立映射关系，由这种思想而构成的数据结构称为 哈希表（散列表）

哈希的本质是用哈希函数将 key 转换为哈希值，以哈希值为索引快速定位存储位置

map&unordered_map

在 C++ 中， 是标准模板库（STL）的一部分，提供了一种基于哈希表的键值对容器。与 std::map 不同，unordered_map 不保证元素的排序，但通常提供更快的查找速度。

特性 map unordered_map
有序性 有序（默认升序，可自定义排序） 无序
速度 插入 / 查找 / 删除 O (log n) 平均 O (1)，最坏 O (n)
内存占用 较小（红黑树结构） 较大（哈希表需要预留空间）
键的要求 支持比较运算符（如 <） 支持哈希函数 + 相等判断
迭代器稳定性 插入 / 删除不失效（除被删元素） 插入可能触发扩容，迭代器失效

std::map 是基于红黑树实现的有序容器，它要求 key 类型满足：

  • 支持 < 比较运算符（默认），或你自定义比较规则；
  • 必须是可拷贝 / 可移动的类型（C++ 基本类型、自定义类都满足）。

常见的合法 key 类型：

  • 数字（int、float、double 等）；
  • 字符串（std::string，核心库已实现 < 运算符）；
  • 自定义结构体 / 类（需手动实现 < 运算符）；
  • 指针、枚举、pair 等（只要支持 < 比较）。

std::unordered_map 是基于哈希表实现的无序容器，它要求 key 类型满足：

  • 有对应的 哈希函数（能计算唯一的哈希值）；
  • 支持 == 相等运算符（用于哈希冲突时比较）；
  • 可拷贝 / 可移动。

常见的合法 key 类型：

  • 数字（int、float 等，标准库提供了哈希函数）；
  • std::string（标准库提供了哈希函数）；
  • 自定义类型（需手动实现哈希函数和 == 运算符）；
  • ❌ 注意：普通指针可以，但 std::vector、std::list 等容器默认没有哈希函数，不能直接作为 key（需自定义）。

  • 用map的场景：

    1. 需要键值对有序（如按键升序/降序遍历）；

    2. 对最坏时间复杂度有要求（O(log n) 比 O(n) 更稳定）；

    3. 处理自定义类型作为键（只需重载<运算符）。

  • 用unordered_map的场景：

    1. 追求极致的查找/插入速度（高频查询场景）；

    2. 不关心元素顺序；

    3. 数据量较大，需要高效的平均时间复杂度。
